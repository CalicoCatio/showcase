Showcase v4 is the fourth iteration of my Showcase site, featuring many improvements, both ones visible to you and ones behind the scenes, allowing me to better maintain this site, while featuring more information about each project that is being showcased. v4 has had refactors for much of the codebase that was ported from v3, new additions not seen before in v3, as well as a complete visual overhaul. v4 couldn't have been just another update to v3 due to the different ways the websites fundamentally function. v3 did not have any support for screen readers or other accesibility features; v4 doesn't explicitly support screen readers or other accesibility tools, but support should be much better on v4. v3 also did not follow the standard convention for naming custom classes, as well as having many different 'unconventional' solutions to certain 'quirks' with how the page was rendered.

Showcase v4 was made with:
  • Languages: HTML 5, CSS and JS.
  • Engine/Framework: Bootstrap 5.3, a front end framework originally developed for twitter.

Design was thought up by:
  • Ariel / CalicoCatio
The design from v4 is visually very similar to v3, down to the position of the carousels inside each showcase. Behind the scenes, however, the site is radically different. v3 had a focus on making everything as custom as possible, to avoid looking like just a template with the words filled in, even if it meant a lesser experience for the user. v4 is designed to instead make use of Bootstrap as much as possible, only using custom classes and CSS styles when absolutly necessary to reduce bloat and complexity.

Code was written by:
  • Ariel / CalicoCatio
Just like the design, the code for v4 is designed to be as modular as possible. In v3, only the version number was managed by a JS function, in v4 the entire footer, as well as the header/navbar and the breadcrumbs are added to the page with JS. This allows me to modify the design of the footer once, and have it replicated to all of the pages where a footer is present, which makes it much easier to maintain. v4 is also coded in VS 2022, whereas v3 was coded entirely inside of GitHub's online code editor, and yes, it was just as terrible as you might expect. Yes, VS Code would be even better, but I can't be bothered to learn how to use it properly, as well as integrate into with GitHub's version control.

Art was drawn up by:
  • Ariel / CalicoCatio
The only real 'art' in v4 comes as a port from v2, the orange that I use as a favicon. I made this in Paint 3D by using a touchscreen to draw the A over and over again, until I got one that looked acceptable. Since then, the favicon has recived a little TLC as the move from v2 to v3 caused a bit of the image to get cropped off; with v4, I have re-added the missing pixels.

  • When porting some of the CSS from v3 to v4 I noticed that one of the styles restricted the height of the body element, to a maximum of 100%, or one screen height worth of pixels. This caused a lot of problems when I was working on v3, since the body needs to be able to expand past that and I somehow never noticed it.
